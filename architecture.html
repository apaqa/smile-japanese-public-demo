正式版執行架構說明

這個 GitHub Pages 版本是可操作展示網站，用來確認視覺、內容與流程；正式上線會用 WordPress / LMS / WooCommerce 與正式主機、網域、金流資料重建或落地。

展示站不含真金流、真資料庫、真 API。
01

展示網站

目前版本部署在 GitHub Pages，使用 HTML、CSS、JavaScript 與 localStorage 模擬會員、課程、商品、購物車與訂單流程。

  • 方便客戶用手機與桌機直接看流程
  • 後續可替換 Logo、老師照片、課程影片、商品圖與文案
  • 不放密碼、API key、個資、正式金流資料
02

第一版網站落地

正式版會以 WordPress 作為網站核心，保留本展示站確認過的資訊架構與視覺方向，再接上可營運的後台與資料管理。

  • WordPress 管理頁面、品牌內容與基本會員入口
  • LMS 管理課程、章節、會員學習進度
  • WooCommerce 管理商品、購物車、訂單
03

金流與發票

實際金流商與電子發票串接，需要等客戶提供正式文件、測試帳號與欄位規格後再進行 API 開發與測試。

  • 付款方式與訂單狀態會依正式 API 調整
  • 電子發票欄位需確認公司戶、會員載具與捐贈流程
  • 正式刷卡資料不會放在展示站或 GitHub Pages
04

後續擴充

多教師分潤、點數折抵、複雜預約、改期、退費規則會列為後續擴充，避免第一階段上線版一次承擔過重流程。

  • 日語老師先完整，韓語與英語保留分類擴充
  • 進階預約規則可在第一版網站穩定後追加
  • 報表、分潤與優惠機制可依實際營運資料設計

素材替換方式

方便迭代
  • 標品牌 Logo 與色彩客戶確認後可替換 favicon、品牌字與主色系可換
  • 師老師照片與介紹先以展示照片佔位，正式版改用授權照片可換
  • 課課程影片與章節展示站使用 placeholder，正式版接 LMS 課程章節可換
  • 物商品圖片與售價目前是展示商品，正式上線接 WooCommerce 商品資料可換

展示站限制

這個版本的價值是讓客戶確認流程與畫面，不代表正式營運系統已部署。

  • localStorage 只存在單一瀏覽器
  • 清除瀏覽器資料後 demo 狀態會重置
  • 沒有真實付款、發票、簡訊、Email 寄送
  • 正式資料庫與權限控管會在 WordPress 版本實作
